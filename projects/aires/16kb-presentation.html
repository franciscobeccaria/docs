1 · The Problem

Why this work exists

16 KB

Google Play now requires apps targeting Android 15+ to support 16 KB memory pages — native libraries must be aligned to 16 KB, not 4 KB.

Blocked

Without it: no app updates (bug fixes, security, features) and crashes on the newest Android devices.

RN core

The misaligned files live inside Hermes & React Native binaries — no Gradle flag or local patch fixes it.

The only fix is upgrading the React Native runtime + Android NDK toolchain. Deadline was extended to Feb 2027, so the mandate was: do it conservatively — the smallest change that meets the requirement.

Hard constraint: keep Old Architecture (newArchEnabled=false) No New Architecture — too risky, out of scope

3 · The Decision

Why 0.77.3 — verified against the source

"With the 0.77 release, React Native is ready to fully support 16 KB page size." — Official React Native 0.77 release blog (primary source)
  • ✓
    0.77 is the official 16 KB floor. The first release that fully supports it.
  • ✓
    React 19 ships in 0.78, not 0.77. Stopping at 0.77 keeps React 18 → avoids a large, risky JS migration.
  • ✓
    0.82 ignores newArchEnabled=false. So 0.81 is the last Old-Arch-capable release — relevant only to optional Phase 2.
  • ✓
    Google's docs: 16 KB-by-default needs AGP 8.5.1+ & NDK r28+; gated on targetSdk 35 — which AIRES already had.

Phase 1 → 0.77.3 (mandatory).   Phase 2 → 0.81.x (optional, separate decision).

4 · The Discovery

The real cost wasn't React Native core

No analysis predicted this — because none built the actual app. The first Android build's audit came back 18 / 19 aligned: one third-party library shipped a 4 KB binary. Then a pattern emerged.

LibraryAndroid needsiOS (Fabric) needsResolved to
react-native-pdf≥ 7 (alignment)—^7.0.1
react-native-svg≥ 15 (new API)< 15.13 (older Yoga)15.12.1 pinned
react-native-reanimated—3.16.x (3.19 = RN 0.78+ API)~3.16.7
react-native-share—≥ 12 (codegen header)^12.3.1

Android wanted some libraries newer; iOS wanted those same libraries older. Discoverable only by building both platforms.

The version is the means — the build + alignment audit is the gate.

8 · Decision for this meeting

Phase 2 — optional, your call

Recommended path

Ship Phase 1 first

RN 0.77.3 already meets the contractual requirement. Get it through CI + QA and released. Lowest risk, fastest compliance.

Optional modernization

Phase 2 → RN 0.81.x

The last Old-Arch release. Not required for compliance. Crosses the React 18→19 wall at 0.78 and forces bigger migrations:

  • •
    react-query → @tanstack/react-query v5
  • •
    react-navigation v6 → v7 · Firebase v20 → v22

Phase 2 is "0.81 or nothing" — there's no safe middle stop, and only after Phase 1 ships.
