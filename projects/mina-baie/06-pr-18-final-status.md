## 1. Scope recap

### What this PR fixes

The double-button bug originally reported by Pedro: on sold-out variants, both "Add to Bag" and "Join the Waitlist" were rendering simultaneously.

**Root cause:** Race condition between our `DOMContentLoaded` JS and Klaviyo's `onsite.js`. Our code uses FA metafields (`futurestock` / `onhandstock`) as the source of truth for button state. Klaviyo independently checks Shopify's native `variant.available` field and forces `display:block` on `.klaviyo-bis-trigger` elements when it disagrees. Because the two systems can disagree, Klaviyo would win the race condition and show both buttons.

The bug was intermittent: users with adblockers that block Klaviyo never saw it.

### What this PR does NOT fix (out of scope)

* The "Add to Cart" button replacement caused by the **Easy Bundle Builder** app. See section 4 below for full diagnosis. This bug exists in production as well — not a regression from PR #18.

## 2. Final fix applied

### Pattern: wrapper-based isolation

Instead of letting Klaviyo mutate `.klaviyo-bis-trigger` directly, we wrap both buttons (ATC submit + waitlist) in their own `<div>` wrappers. Our JS and Liquid control the wrapper's inline `display` style. Because CSS guarantees that a parent with `display: none` hides all children regardless of child inline styles, Klaviyo can no longer win the race — we always have the last word at the parent level.

### Files changed

| File | Change |
| -- | -- |
| `snippets/buy-buttons.liquid` | Wrapped submit `<button>` in `.atc-btn-wrapper` with `style="{{ atc_display }}"`. Wrapped waitlist `<button>` in `.waitlist-btn-wrapper` with `style="{{ waitlist_button_display }}"`. Updated JS `getElements()` to read/write wrapper visibility instead of button visibility. |
| `snippets/product-sticky-add-to-cart.liquid` | Same wrapper pattern applied to the mobile sticky waitlist button. Moved `sm:hidden` responsive class from button to wrapper. |
| `assets/theme.css` | Added 3 rules (~lines 10723–10738): (a) `.atc-btn-wrapper` and `.waitlist-btn-wrapper` get `width: 100%`; (b) `.waitlist-btn-wrapper .klaviyo-bis-trigger` forced to `display: block !important; width: 100%`; (c) `product-form .klaviyo-bis-trigger:not(.klaviyo-bis-manual-btn)` hidden. |

### Liquid state machine (unchanged from PR #9, restated for clarity)

```
if variant == nil                                  → Unavailable (disabled)
elsif futurestock > 0 and onhandstock <= 0         → Pre-Order  (submit enabled, waitlist hidden)
elsif variant.available == false                   → Sold Out   (submit hidden, waitlist shown)
else                                               → In Stock   (submit enabled, waitlist hidden)
```

JS mirrors the same states in `applyState()` and runs on: `change` events (variant/quantity), `cart:change`, `cart:refresh`, form `submit`, and the initial post-DOMContentLoaded sync.

## 4. Out-of-scope finding: Easy Bundle Builder replaces ATC button

### Symptom

QA reported that when switching between variants on some products, the "Add to Bag" button was replaced by "Add to Cart" (different casing, different styles). Intermittent.

### Diagnosis

The offending button in the DOM:

```html
<button type="submit"
        class="product-form__buttons-add-to-cart button w-full pn-button-o5fnyp0twoh"
        value="Add to Cart"
        title="">Add to Cart</button>
<div class="gbbExtBundleUpsellWrapper"></div>
```

Three tells that this is not our theme's render:

1. `gbbExtBundleUpsellWrapper` sibling div — the `gbb` prefix is from **Easy Bundle Builder** (Shopify app).
2. `pn-button-o5fnyp0twoh` — hashed class name generated by the app.
3. **Missing** `<button-content class="button__content">` wrapper — our `snippets/button.liquid` always wraps the text in this custom element.

Confirmed in `config/settings_data.json:110`:

```json
"type": "shopify://apps/eb-easy-bundle-builder/blocks/app-embed/05b1325c-6303-4da5-8e2f-b13ab2a50e1a"
```

Easy Bundle Builder is installed as an **app-embed**, which Shopify loads globally on all templates. On variant change, the app replaces the ATC button with its own.

### Why it's not a PR #18 regression

Confirmed by QA: the same behavior happens on production before this branch.

### Recommended next steps (separate ticket)

1. **Admin config:** Shopify Admin → Apps → Easy Bundle Builder — if bundles aren't actively used, disable the app-embed via Online Store → Themes → Customize → App embeds.
2. **Per-product exclusion:** Most bundle apps allow an allowlist of products where the button is NOT replaced.
3. **Theme hack (last resort):** MutationObserver — not recommended, fragile.

**Escalate to:** Pedro / b.solutions.
